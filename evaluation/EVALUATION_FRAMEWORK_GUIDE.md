# MCQ Quality Evaluation Framework - Complete Guide

## Overview

This framework evaluates MCQs (Multiple-Choice Questions) generated by LLMs using a hybrid approach combining:

1. **Item-Writing Flaws (IWFs)** - 10 objective rules (yes/no checklist)
2. **Semantic Quality** - 3 Likert-scale dimensions (1-5 scoring)

**References:**
- Arif et al. (2024) "Generation and Assessment of MCQs from LLMs" — L@S '24
- Elkins et al. (2024) — AAAI '24
- Zheng et al. (2023) "Judging LLM-as-a-Judge" — addresses cross-model evaluation bias

---

## 1. Item-Writing Flaws (IWFs) - 10 Critical Rules

Each flaw is evaluated as **YES** (flaw present) or **NO** (flaw absent).

**Pass Rate = (10 - number of flaws) / 10**

### Flaw Categories

#### Group 1: Stem (Question) Issues

1. **Unfocused Stem**
   - Does the question lack a clear, single focus?
   - Example ❌: "What can you tell me about databases and SQL?"
   - Example ✓: "What is the primary function of a primary key in relational databases?"

2. **Unclear Information**
   - Is any information in the stem ambiguous or confusing?
   - Example ❌: "The thing about the data with the structure is..."
   - Example ✓: "Database normalization is the process of organizing data to minimize redundancy."

3. **Negative Wording Without Emphasis**
   - Uses negation (NOT, EXCEPT, NEVER) without highlighting those critical words?
   - Example ❌: "Which is not a type of normalization?" (NOT is not highlighted)
   - Example ✓: "Which of the following is NOT a type of normalization?" (stronger, but should highlight NOT)

#### Group 2: Options (Answers) Issues

4. **Implausible Distractors**
   - Are any distractors obviously wrong or unrealistic?
   - Example ❌: "Which is a database? (A) SQL, (B) Banana, (C) Oracle, (D) PostgreSQL"
   - Example ✓: "Which is a database? (A) SQL, (B) MySQL, (C) T-SQL, (D) Pig Latin"

5. **Heterogeneous Options**
   - Are the 4 options inconsistent in type, length, or grammatical structure?
   - Example ❌: 
     - A) Database
     - B) A structured collection of related data
     - C) MySQL
     - D) Tables
   - Example ✓:
     - A) Foreign key
     - B) Primary key
     - C) Composite key
     - D) Surrogate key

6. **Longest-Answer-Correct Bias**
   - Is the correct answer noticeably longer than distractors?
   - Example ❌:
     - A) Data
     - B) Information
     - C) A systematically organized collection of related data stored in one place with controlled access, designed for efficient retrieval
     - D) Table
   - Example ✓: All options similar length

7. **Word Repeats**
   - Do any options share distinctive/unique words with the stem (giveaway)?
   - Example ❌: Stem: "...ACID properties..." → Distractor: "ACID compliance" (too obvious)
   - Example ✓: Stem uses concept words, options use different language

8. **Absolute Terms in Distractors**
   - Do distractors contain absolute language (always, never, all, none)?
   - Example ❌: Distractors: "Databases always...", "Never use..."
   - Example ✓: Distractors use qualified language: "Databases typically...", "Most databases..."

#### Group 3: Content Issues

9. **Multiple Correct Answers**
   - Could more than one option reasonably be correct?
   - Example ❌: Q: "What is a database?" A: (A) Organized data, B: Store of information (Both correct!)
   - Example ✓: Only one answer is definitively correct

10. **Factual Error**
    - Is any information factually incorrect based on source material?
    - Example ❌: "SQL was invented in 1995" (Actually 1974)
    - Example ✓: All facts verified against source

## 2. Semantic Quality - 3 Likert Dimensions (1-5 Scale)

Measures meaning and alignment. **Average Score = (relevance + answerability + bloom) / 3**

### Dimension 1: Relevance
**How well does the question relate to and test the source content?**

- **5**: Question directly tests a critical concept from the source; essential for understanding
- **4**: Tests an important concept from the source; key knowledge
- **3**: Tests a minor or peripheral concept; somewhat related
- **2**: Tangentially related; requires external knowledge
- **1**: Not related to source OR contains hallucination

**Example Scoring:**
- ✓ 5: Source discusses "ACID properties" → Q asks to identify which property ensures durability
- ✓ 3: Source discusses databases broadly → Q asks about spreadsheet features
- ❌ 1: Source about relational databases → Q asks about blockchain

### Dimension 2: Answerability
**Can the question be confidently answered using ONLY the source material?**

- **5**: Clearly and completely answerable from source; all info provided
- **4**: Mostly answerable from source; minor gaps acceptable
- **3**: Requires some inference or general knowledge beyond source
- **2**: Significant information gaps; source insufficient
- **1**: Cannot be answered from source alone; requires external expertise

**Example Scoring:**
- ✓ 5: Source explains "Primary keys uniquely identify records" → Q asks "What uniquely identifies each record?"
- ✓ 3: Source explains normalization → Q asks to apply normalization to a new scenario
- ❌ 1: Source about SQL basics → Q asks "Which NoSQL database is best for healthcare?"

### Dimension 3: Bloom Alignment
**Does the question match the labeled Bloom level?**

Bloom's Taxonomy Levels (in order):
1. **Remember**: Recall facts (define, list, state)
2. **Understand**: Explain meaning (describe, explain, summarize)
3. **Apply**: Use knowledge in new situations (apply, solve, use)
4. **Analyze**: Break down concepts (analyze, compare, distinguish)
5. **Evaluate**: Make judgments (judge, critique, evaluate)
6. **Create**: Produce something new (design, create, develop)

**Scoring:**
- **5**: Perfect alignment (question operationalizes the exact Bloom level)
- **4**: Mostly aligned (±0 levels error)
- **3**: Slightly misaligned (±1 level error)
- **2**: Notably misaligned (±2 levels error)
- **1**: Completely wrong (±3+ levels error)

**Examples:**
- ✓ Labeled "Remember" + Q: "What is database normalization?"
- ✓ Labeled "Apply" + Q: "Apply 3NF rules to this schema..."
- ❌ Labeled "Apply" + Q: "Define database normalization" (Actually Remember)

## 3. Overall Verdict Rules

### PASS ✓
- **IWF Pass Rate ≥ 80%** (max 2 flaws)
- **AND Semantic Average ≥ 4.0** (strong quality across 3 dimensions)
- **Status**: Production-ready, can use immediately

### BORDERLINE ⚠
- **IWF Pass Rate ≥ 70%** (max 3 flaws)
- **AND Semantic Average ≥ 3.5**
- **Status**: Acceptable with minor revisions

### FAIL ❌
- **Below borderline thresholds**
- **Status**: Requires significant revision

## 4. Usage Instructions

### 4.1 Basic Single Evaluation

```python
from evaluation.mcq_evaluator import (
    MCQEvaluation, IWFFlaws, SemanticQuality, MCQEvaluator
)

# Create flaws checklist (findings from LLM judge)
flaws = IWFFlaws(
    unfocused_stem=False,
    unclear_information=False,
    # ... (set to True if flaw is present)
)

# Create semantic scores (1-5 for each dimension)
semantic = SemanticQuality(
    relevance=5,
    answerability=4,
    bloom_alignment=5
)

# Create evaluation
eval = MCQEvaluation(
    question_id="Q001",
    question_text="What is a primary key?",
    correct_answer="A unique identifier for each record",
    bloom_level="Understand",
    flaws=flaws,
    semantic=semantic,
    feedback="Clear question, well-aligned with Understand level"
)

# Check results
print(f"IWF Pass Rate: {eval.flaw_pass_rate:.1%}")  # 90%
print(f"Semantic Average: {eval.semantic_average:.2f}")  # 4.67
print(f"Verdict: {eval.overall_verdict}")  # "pass"
```

### 4.2 Batch Evaluation & Reporting

```python
evaluator = MCQEvaluator(judge_model="gpt-4-turbo")

# Add evaluations

evaluator.batch_evaluate(evaluations)
```

## 5. LLM Judge Prompt

### How to Use the Prompt

1. **Extract source chunk** from your training document
2. **Get generated MCQ** from Gemini or your LLM generator
3. **Send to judge LLM** (GPT-4, Claude, etc.) with the evaluation prompt
4. **Parse JSON response** into MCQEvaluation object

### Judge Model Recommendation

**Cross-Model Setup (RECOMMENDED):**
- **Generator**: Gemini (fast, cheap generation)
- **Judge**: GPT-4 or Claude (avoids self-evaluation bias)

**Why avoid self-evaluation?**
- Zheng et al. (2023) showed LLMs have inherent bias toward their own outputs
- GPT-4 judging GPT-4 → inflated scores
- Gemini judging Gemini → inflated scores
- Mixing models → more objective evaluation

### Prompt Template

See `evaluation/mcq_evaluator.py` → `EVALUATION_PROMPT_TEMPLATE`

Example integration:

```python
from evaluation.mcq_evaluator import EVALUATION_PROMPT_TEMPLATE
import openai

# 1. Format prompt
prompt = EVALUATION_PROMPT_TEMPLATE.format(
    chunk_text=source_content,
    question=mcq.question,
    option_a=mcq.options[0],
    option_b=mcq.options[1],
    option_c=mcq.options[2],
    option_d=mcq.options[3],
    correct_answer=mcq.correct_answer,
    bloom_level=mcq.bloom_level
)

# 2. Call judge LLM
response = openai.ChatCompletion.create(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": prompt}],
    temperature=0.0  # Deterministic evaluation
)

# 3. Parse JSON response
import json
result = json.loads(response["choices"][0]["message"]["content"])

# 4. Create MCQEvaluation
eval = MCQEvaluation(
    question_id=mcq.id,
    question_text=mcq.question,
    correct_answer=mcq.correct_answer,
    bloom_level=mcq.bloom_level,
    flaws=IWFFlaws(**result["flaws"]),
    semantic=SemanticQuality(**result["semantic"]),
    feedback=result["feedback"]
)
```

## 6. Report Interpretation

### Key Metrics for Thesis

**In Experiments section, report:**

1. **Production-Ready Rate**
   - % MCQs with Pass verdict
   - Target: ≥ 70% for graduation thesis
   - Report: "312/450 generated MCQs (69.3%) passed quality standards"

2. **Top 3 Flaws Distribution**
   - Shows system weaknesses
   - Suggests future improvements
   - Report: "Most common flaw was implausible_distractors (18%), suggesting..."

3. **Semantic Averages by Dimension**
   - Shows which aspect needs work
   - Report:
     ```
     Semantic Quality (1-5 scale):
     - Relevance: 4.2 (strong connection to source)
     - Answerability: 3.8 (requires minor inference)
     - Bloom Alignment: 4.5 (good level matching)
     - Overall: 4.17/5
     ```

4. **Performance by Bloom Level**
   - Different difficulty zones perform differently
   - Report table:
     ```
     | Bloom Level | Count | Pass Rate | Semantic Avg |
     |---|---|---|---|
     | Remember | 80 | 82% | 4.3 |
     | Understand | 150 | 71% | 4.1 |
     | Apply | 120 | 58% | 3.9 |
     | Analyze | 70 | 45% | 3.6 |
     | Evaluate/Create | 30 | 27% | 3.2 |
     ```

5. **Correlation Analysis** (Optional but impressive)
   - Calculate correlation between IWF Pass Rate and Semantic Average
   - If r > 0.7: metrics are redundant (pick one)
   - If r < 0.5: metrics measure different aspects (both valuable)
   - Report: "IWF Pass Rate and Semantic Average showed weak correlation (r=0.42), indicating they measure distinct quality dimensions"

## 7. Troubleshooting

### Problem: All MCQs get PASS verdict
- LLM judge too lenient
- Solution: Lower thresholds in code or use stricter judge

### Problem: All MCQs get FAIL verdict
- Thresholds too strict
- Solution: Review failure patterns, adjust rubric

### Problem: Inconsistent scores from judge
- Temperature too high (use 0.0 for evaluation)
- Judge model too weak (use GPT-4 or Claude)
- Prompt unclear (refine evaluation prompt)

### Problem: JSON parsing errors
- Judge LLM returned markdown code block
- Solution: Add `json.loads()` error handling:
  ```python
  try:
      result = json.loads(response)
  except:
      # Extract JSON from markdown if wrapped in ```json ... ```
      content = response.split("```json")[1].split("```")[0].strip()
      result = json.loads(content)
  ```
